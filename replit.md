# Overview
Educafric is a mobile-first, bilingual Progressive Web App (PWA) EdTech platform designed for African educational institutions. Its primary goal is to integrate academic administration, communication, and financial management using a Competency-Based Approach (CBA). The project aims to deliver user-friendly, scalable, and culturally relevant EdTech solutions to expand market reach and enhance access to quality education across Africa.

# System Architecture
- **UI/UX Decisions**: Mobile-first PWA design with a modern purple theme (`#7C5CFC`) leveraging Radix UI, Shadcn/UI, and Tailwind CSS. Alert dialogs maintain a consistent white background. Student ID cards adhere to an official template, supporting mobile-friendly color printing, digital signatures, and QR verification. Online course interfaces are always visible, featuring dynamic access rules and contextual prompts for restricted functionalities. Responsive tabs adapt for mobile (icons only) and desktop (icons with text). Dashboard titles are centrally managed by `DashboardNavbar` to prevent duplication, and module icons retain their distinct original colors. The Android application restricts access for Commercial, SiteAdmin, and Admin profiles, providing bilingual error messages.
- **Technical Implementations**: The frontend is built with React (TypeScript), Wouter for routing, and TanStack Query for data management and PWA capabilities. The backend is an Express.js-based RESTful API. Data persistence uses PostgreSQL on Neon Serverless, managed with Drizzle ORM. Authentication is session-based with `express-session`, `Passport.js`, and Firebase Google OAuth, featuring an 8-role-based access control (RBAC) model. Phone numbers serve as primary unique identifiers. The route architecture prioritizes internal routes and ensures strict segmentation of module mappings per dashboard. `queryClient.ts` uses `serializeQueryKey()` for immediate data display and cache invalidation. A centralized document generation system (`server/services/pdfGenerator.ts`) ensures instant document creation, storing files in `/public/documents`. Homework attachments are supported via dedicated upload endpoints and JSONB storage. Academic bulletins are generated using an adaptive flexbox layout defined in `client/src/index.css`, optimized for print and `html2canvas`, dynamically incorporating school logos via a 4-level fallback for PNG/JPEG formats. Android stability is enhanced with `GlobalErrorBoundary.tsx` for error capture, `networkMonitor.ts` for connectivity, and `queryClient.ts` for intelligent retry logic.
- **Feature Specifications**: Real-time attendance tracking with 'Present', 'Late', 'Absent' statuses, flexible timetable management, and multi-channel notifications (Email, WhatsApp, PWA). It supports bilingual templates and robust bulk Excel import with immediate data display. Key academic features include CBA bulletin generation and Jitsi Meet integration for online classes. Schools can define custom academic levels via API and UI. Dynamic online course access rules are implemented based on teacher subscription status. Configurable parent subscription plans and automatic family discounts are managed by Site Administrators. Student messaging is restricted to parents for sending, while receiving from all other profiles is permitted. Automated homework creation and submission notifications, with scheduled reminders, with a comprehensive student-parent connection workflow ensures secure linkages. Hierarchical archives provide organized access to past academic documents. Student photos are comprehensively mapped across various field aliases. Competencies are automatically imported into bulletins from the `competencies` table.
- **System Design Choices**: All storage modules exclusively use Drizzle ORM for database queries, prohibiting hardcoded/mock data. Strict multi-tenancy is enforced with `user.schoolId` for data isolation. Sandbox data is pre-seeded by idempotent scripts for consistent demo environments. A multi-role user management system (`role_affiliations`) includes existing user detection by email/phone. Development and production databases are completely separated. All "Contacter Educafric" buttons open WhatsApp with pre-filled messages to the official number `237656200472`. Bulk teacher import uses a default password (`Educafric2024!`), assigns the 'Teacher' role, and employs `ON CONFLICT DO NOTHING` for deduplication. The `enrollments` table is designated for student class registrations, and `parent_student_relations` manages parent-student linkages. Parent API responses consistently return objects containing the data, requiring explicit extraction on the frontend. Student photo fields are comprehensively mapped across various aliases for broad frontend compatibility. Competencies are automatically imported into bulletins from the `competencies` table. Subject synchronization across modules is handled via `PUT /api/classes/:classId`, with duplicate detection and extensive cache invalidation. Approved grades for multiple subjects are merged for bulletin display.
